## **Extended Q&A Preparation**

### **Technical Questions**

**Q**: *"How exactly do you extract deterministic signals?"*  
**A**: *"We analyze file metadata using standard tools - file size, estimated bitrate from size/duration ratios, MD5 hash prefixes for complexity scoring, and regex pattern matching on filenames. These never change for the same file."*

**Q**: *"What if two different videos have similar characteristics?"*  
**A**: *"That's actually good - videos with similar characteristics should route similarly. The key is that the SAME video always routes identically. Different videos with similar compression levels should both go to the compression model."*

**Q**: *"How do you handle edge cases in routing?"*  
**A**: *"Our policy engine has fallback rules. Every video gets the baseline model at minimum. Additional specialists are added based on detected characteristics. If no special characteristics are detected, only the baseline runs - which is deterministic behavior."*

### **Forensic/Government Questions**

**Q**: *"Can this be used in court?"*  
**A**: *"Yes - the routing decisions are now fully deterministic and auditable. We provide complete documentation of why each model was selected, based on immutable file characteristics. Same evidence file will always produce same routing decision."*

**Q**: *"How do you ensure no bias in routing?"*  
**A**: *"Routing is based purely on technical file characteristics, not content analysis. The bias is explicit and documented - compressed videos go to compression specialists, low-light videos go to lighting specialists. This is transparent and auditable."*

**Q**: *"What about adversarial attacks on routing?"*  
**A**: *"An attacker would need to modify fundamental file characteristics like size, format, or encoding. But these modifications would be detectable and actually help our analysis - if someone re-encodes a video to avoid compression detection, the re-encoding itself becomes evidence."*

### **Innovation Questions**

**Q**: *"Is this still innovative if it's rule-based?"*  
**A**: *"Absolutely. The innovation is in the intelligent specialist selection and ensemble approach. We're the first to use deterministic file-characteristic routing for deepfake detection. The agentic intelligence is in knowing which specialists to combine, not in stochastic routing."*

**Q**: *"How does this compare to competitors?"*  
**A**: *"Competitors use single-model approaches or simple ensembles. We're the only system with intelligent routing to specialized models. Making that routing deterministic actually makes us more advanced - we have the intelligence without the unpredictability."*
